= 1.2.0 — 2026 =
* Sistema de actualizaciones definido: el core de Ultron se actualiza siempre de forma manual (subiendo el zip); los módulos (core y addon) se actualizan desde GitHub leyendo modules.json y comparando versiones.
* Los módulos core (WordPress Monitor, Database Monitor, Storage Monitor, Plugins Monitor) ahora tienen su propio repositorio en GitHub y son actualizables individualmente, aunque siguen viniendo preinstalados en el zip de Ultron.
* Nueva lógica de descarga e instalación de módulos desde su repositorio de GitHub (tags de versión vía la API zipball de GitHub), soportando repos públicos y privados con token.
* Botón "Actualizar" en la pestaña Módulos para cualquier módulo con nueva versión disponible.

= 1.1.0 — 2026 =
* Reorganización del menú: Opciones pasa a ser submenú independiente (antes pestaña del hub).
* Nueva pestaña "Información" en el hub: banner de versión, modo de uso y novedades (changelog).
* Eliminada la configuración de "Ultron Master" de Opciones (se moverá a un módulo de telemetría independiente).
* Limpieza de class-options.php y separación en Ultron_Options_Page.

= 1.0.0 — 2025 =
* Versión inicial de Ultron.
* Hub principal con pestañas Dashboard, Módulos y Opciones.
* Sistema de módulos activables/desactivables individualmente.
* Submenú Monitor compartido con orden fijo de pestañas.
* Módulo WordPress Monitor: snapshot de instalación, servidor, configuración, seguridad, archivos expuestos, endpoints y usuarios.
* Módulo Database Monitor: listado de tablas con estado de salud y truncate seguro con doble confirmación.
* Módulo Storage Monitor: inodos y tamaño de instalación y uploads, desglose por año/mes, vigilancia de error.log.
* Módulo Plugins Monitor: detección de plugins recomendados, conflictos, licencias premium y acciones rápidas.
* Opciones: conexión con Ultron Master, token de GitHub, límites de histórico, umbral de error.log, actualización automática y desinstalación limpia.
* Assets CSS unificados con sistema de variables y clases consistentes.
